# CPF Documentation

This directory contains comprehensive documentation for the Cybersecurity Psychology Framework (CPF) SOC integration.

## Contents

### Papers & Guides

- **cpf_implementation_guide.md** - Step-by-step implementation guide for SOC integration
- **cpf_technical_integration.md** - Technical details for SIEM/SOAR integration
- **cpf_integration_descriptive.md** - Descriptive overview of the integration architecture
- **cpf_pattern_examples_list.md** - Real-world examples of psychological patterns
- **cpf_psychological_inference_paper.md** - Theoretical foundation for psychological inference
- **cpf_vulnerability_whitepaper.md** - Whitepaper on human vulnerability assessment
- **cpf_enterprise_paper.tex/.pdf** - Enterprise deployment considerations

## Document Organization

### Implementation Guides
Start with `cpf_implementation_guide.md` for practical implementation steps, including:
- Data source configuration
- Pattern detection algorithms
- Indicator scoring methodology
- Dashboard integration

### Technical Documentation
`cpf_technical_integration.md` provides technical specifications:
- API endpoints and data formats
- SIEM correlation rule structure
- Bayesian scoring formulas
- Alert threshold definitions

### Theoretical Foundation
`cpf_psychological_inference_paper.md` explains the psychological theory behind the framework:
- Cognitive vulnerability models
- Behavioral pattern recognition
- Risk quantification methodology

## Related Resources

- **Implementation Files**: `/implementation/` - Detailed OFTLISRV schemas for all 100 indicators
- **Source Code**: `/src/` - Python implementation of pattern detectors
- **Papers**: `/papers/` - Academic papers and technical specifications

## Usage

These documents are designed to be read in sequence:

1. Start with `cpf_implementation_guide.md` for overview
2. Review `cpf_pattern_examples_list.md` for real-world context
3. Consult `cpf_technical_integration.md` for implementation details
4. Reference specific indicator implementations in `/implementation/`
